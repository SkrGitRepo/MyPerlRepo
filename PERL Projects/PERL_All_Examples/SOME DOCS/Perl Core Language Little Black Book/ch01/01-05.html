Using The -w Switch For Warnings

When working with Perl, it’s always a good idea to use the -w switch, and many Perl stylists are fanatical about this. I recommend it as well.

The -w switch warns about many things, including:

•  Variable names mentioned only once
•  Scalar variables (such as simple variables) that are used before being set
•  Redefined subroutines
•  References to undefined file handles (you work with files using file handles in Perl; see the next topic)
•  File handles opened as read-only that you attempt to write to
•  Values used as a number that don’t look like numbers
•  Using an array as though it were a scalar variable
•  Subroutines that use recursion more than 100 levels deep

Handling Text Input And Output With Standard File Handles

Perl treats input and output as channels, and you work with these channels using file handles. A file handle is just a value that represents a file to Perl, and you get a file handle for a file when you open that file.

You can use three predefined file handles with text:

•  STDIN—The normal input channel for your script
•  STDOUT—The normal output channel
•  STDERR—The normal output channel for errors

By default, these file handles correspond to your terminal. We’ll use predefined file handles in this chapter, as in the next topic, in which we use STDOUT to display text.

Displaying Text

To print text to a file, including STDOUT, use the print statement, which has these forms:

print FILEHANDLE LIST
print LIST
print

If you don’t specify a file handle, STDOUT will be used. If you don’t specify a list of items to print (a list may be made up of only one item), the print function prints whatever is in the special Perl variable $_ to the output channel. $_ is the default variable that holds input from the input channel (see “Reading Typed Input” later in this chapter for more information).

Here’s an example in which we just print “Hello!” and a newline character to the output channel:

print “Hello!\n”;

Hello!

The print function is really a list function that takes a list of items (we’ll cover Perl lists in the next chapter). This means you can pass a list of items to print like this, in which we separate the list items with commas:

print “Hello ”, “there!\n”;

Hello there!

Note that it’s also possible to get very fancy when you print text in Perl because you can use Perl formats and the formatted print function, printf, as we’ll see in Chapter 11.

Displaying Line Numbers And Script File Names

You can display the current line of execution in a Perl script by referring to it with the _ _LINE_ _ token and the name of the current file with the _ _FILE_ _ token. For example, in this one-line script, you can see that the current line is line number 1:

%perl -e “print _ _LINE_ _;”

1

Printing Text A Number Of Times

If you want to print a string a number of times, use the Perl x repetition operator like this:

print “Hello!\n” x 5; Hello!

Hello!
Hello!
Hello!
Hello!

Or like this if you want to draw a horizontal line made up of hyphens: 

print “-” x 30

Basic Text Formatting

You can perform some basic test formatting using escape characters in Perl. An escape character is a special character you preface with a backslash (\), and it, too, indicates functions like print that handle text strings. The escape character is a special character that should be specially interpreted. Some of the escape characters and what they represent appear in Table 1.1.

For example, here’s how you can display double quotes in a printed string:

print “\”Hello!\“\n”;


“Hello!”

Here’s how to use tabs: 

print “Hello\tfrom\tPerl.\n”;

Hello   from   Perl.

Here’s how to create multiline output using the \n newline character:

print “Hello\nfrom\nPerl.\n”;

Hello
from
Perl.

Keep in mind that this is just basic text formatting and that much more complex formatting is possible. There’s always another way of working with text in Perl—after all, it was created as a text-handling language. For example, see the next topic on Perl here documents.

Table 1.1 Escape characters and what they represent.
Escape Character Means
\“ Double quote
\t Tab
\n Newline
\r Return
\f Form feed
\b Backspace
\a Alarm (bell)
\e Escape
\033 Octal character
\x1b Hex character
\c[ Control character
