Writing Perl Code: Statements And Declarations

Perl code is made up of statements and declarations. Declarations indicate to Perl how you intend to use specific programming constructs before you use them, and are only necessary for formats and subroutines, although you can also declare other items like variables. You’ll learn more about declarations later in the book.

Statements come in two forms: simple and compound. A simple statement is an expression that performs some specific action. In code, simple statements end with a semicolon (;) like the following snippet, where we use the print function to display the string Hello!, followed by a newline character, \n (see “Basic Text Formatting” later in this chapter), which skips to the next line on the screen:

print “Hello!\n”;

Compound statements are composed of expressions and blocks. Blocks are delimited with curly braces in Perl—{ and }—and can hold multiple simple statements. Blocks also have their own scope (the scope of items, like a variable, indicate where in the program you can use that variable, as we’ll see later), and you don’t place a semicolon after the curly braces. Here’s an example of a block to create a compound for loop statement, the most basic of the Perl loops (we’ll learn more about the for loop in Chapter 5):

for ($loop_index = 1; $loop_index <=5; $loop_index++) {
     print “Hello”;
     print “there!\n”; }

Running Perl Scripts

Suppose you had a file named hello.pl containing this Perl script: 

#!/usr/local/bin/perl5 -w    #Use Perl 5 with warnings

print “hello\n”;

How do you actually run it? This is a basic Perl step, but because many variations are available, we’ll take a look at them here.

If Your Script Can Find Perl

If your script can find Perl (see “Making Sure Your Script Can Find Perl” earlier in this chapter), you can run the script easily. In Unix, this means that you’ve included a line like #!/usr/local/bin/perl5 -w as the first line in your script. Make the script an executable file with chmod like this:

chmod +x hello.pl

Also, make sure the script is in your path. For example, check your .login file and look for set path commands if your shell is csh or derived from csh. If your shell is sh or derived from sh, check your .profile file for PATH statements. If your shell is bash, check your .bash_profile file for PATH statements, or if you have a different shell, use techniques appropriate for that shell. Then just run the script at the command line like this:

%hello.pl

In Windows and on the Macintosh, just double-click the script file to run it. Make sure you’ve given the script file the extension .pl in Windows, which is the extension that the ActiveState software uses to connect to the Perl interpreter.

In MS-DOS, after you’ve used the pl2bat.bat batch file to convert your script into a BAT file (see “Making Sure Your Script Can Find Perl” earlier in this chapter), just run the BAT file at the DOS prompt like this:

C:\>hello.bat

If You Want To Use Perl From The Command Line

To run a script explicitly with the Perl interpreter, make sure perl is in your path, and use the perl command, which generally looks like the code below. The switches in brackets, [], are optional. See “Using Command Line Switches” later in this chapter to find out what all these switches mean:

perl [ -sTuU ] [ -hv ] [ -V[:configvar] ] [ -cw ]
[ -d[:debugger] ] [ -D[number/list] ] [ -pna ]
[ -Fpattern ] [ -l[octal] ] [ -0[octal] ] [ -Idir ]
[ -m[-]module ] [ -M[-]‘module...’ ] [ -P ] [ -S ]
[ -x[dir] ] [ -i[extension] ] [ -e ‘command’ ]
[ -- ] [ programfile ] [ argument ]...

When you use the perl command this way, Perl looks for your script in one of these places:

•  Line by line with -e switches on the command line.
•  In the file given by the first file name on the command line.
•  Passed in line by line using standard input if you specify a hyphen (-) for the script name.

We’ll take a look at each of these methods here.

Using the -e switch, you can pass code directly to Perl, line by line. (On some systems, you can use multiple -e switches to pass multiple lines of code.) As mentioned in the “Introduction,” we’ll display program output in a bold, italic font to distinguish it from code:

%perl -e “print ‘Hello!\n”;’

Hello!

However, you have to be careful about what kinds of quotes you can use on some systems. Here’s how you would execute the same line in MS-DOS (notice that I’ve replaced the double quotes in the string I want to print with escaped quotes, (\";) (see “Basic Text Formatting” later in this chapter):

C:\>perl -e “print \”Hello!\n\“”;

Hello!

Of course, you can also place your script in a file and pass that file’s name to the Perl interpreter. For example, if this is the content of the file hello.pl (we’re omitting the #! line because we’re running the Perl interpreter explicitly)

print “Hello!\n”;

you could then run that script this way, specifying that file name: 

%perl hello.pl

Hello!

You can also type in a multiline script if you use a hyphen (-) for the script name (this is the default, even if you omit the hyphen and just type “perl”): 

%perl -

In this case, Perl waits for you to type in your complete script: 

%perl -
print “Hello!\n”;

How do you indicate that Perl should execute the script? You type in the __END__ token like this:

%perl -
print “Hello!\n”;
__END__

Hello!

You will execute the whole script at once if you use this method. For testing purposes, you may want to run statements one by one, interactively. To do that, you might develop a Perl mini-shell, as we’ll do in the next topic.

Running Perl Scripts Interactively

For testing purposes, you might want to execute Perl statements interactively, one by one as you type them in, seeing the results immediately. To do that, you can write a Perl shell. Here’s a short working example: 

#!/usr/bin/perl5 -w     # Use Perl 5 with warnings
use strict;             # Require variable declarations, etc.
my $count = 0;          # $count used to match {}, (), etc.
my $statement = “”;     # $statement holds multi-line statements
local $SIG{__WARN__} = sub {}; # Suppress error reporting

while (<>) {        # Accept input from the keyboard

    chomp;              # Clean up input
    while (/{|\(|\[/g) {$count++};    # Watch for {, (, etc.
    while (/}|\)|\]/g) {$count--};    # Pair with }, ), etc.

    $statement .= $_;    # Append input to current statement

    if (!$count) {       # Only evaluate if {, ( matches }, )
        eval $statement; # Evaluate the Perl statement
        if($@) {print “Syntax error.\n”};  # Notify of error
        $statement = “”; # Clear the current statement
        $count = 0       # Clear the multiline {, ( count
     }
}
