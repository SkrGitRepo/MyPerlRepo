Working With Radio Buttons

To create radio buttons, you use the <INPUT> tag with the TYPE attribute set to radio. To group a set of radio buttons, you give them all the same name. Here’s how to create a radio button group in lib1.cgi to let the user indicate the day of the week; note that the first radio button includes the attribute CHECKED, so it appears selected by default:

print
“Please indicate the day of the week: <P>
<INPUT TYPE=\“radio\” NAME=\“radios\” VALUE=\“1\” CHECKED>
Sunday
<INPUT TYPE=\“radio\” NAME=\“radios\” VALUE=\“2\”>Monday
<INPUT TYPE=\“radio\” NAME=\“radios\” VALUE=\“3\”>Tuesday
<INPUT TYPE=\“radio\” NAME=\“radios\” VALUE=\“4\”>Wednesday
<INPUT TYPE=\“radio\” NAME=\“radios\” VALUE=\“5\”>Thursday
<INPUT TYPE=\“radio\” NAME=\“radios\” VALUE=\“6\”>Friday
<INPUT TYPE=\“radio\” NAME=\“radios\” VALUE=\“7\”>Saturday”;

You can see the results of this code in Figure 19.3, shown earlier in the chapter. In lib2.cgi, we determine which radio button the user clicked and display that this way, as shown earlier in Figure 19.4: 

if (&ReadParse(*in)) {
    print
   “Today is day <EM>”, $in{‘radios’},“</EM> of the week.”;
}

Working With Pop-Up Menus

A pop-up menu is just a scrolling list in which only one item is visible, so, like scrolling lists, you create it with the <SELECT> tag, but without specifying the SIZE attribute (which defaults to one). For example, here’s how to create a pop-up menu in lib1.cgi to ask the user how much unsolicited mail he or she wants:

print
“Thank you for filling out our Survey. Please indicate how
much unsolicited mail you like to get:
<SELECT NAME=\“popupmenu\”>
<OPTION  VALUE=\“Very much\”>Very much
<OPTION  VALUE=\“A lot\”>A lot
<OPTION  VALUE=\“Not so much\”>Not so much
<OPTION  VALUE=\“None\”>None
</SELECT><P>”;

You can see the results in Figure 19.3, shown earlier in the chapter. In lib2.cgi, here’s how to display the item the user selected in the pop-up menu, as shown earlier in Figure 19.4: 

if (&ReadParse(*in)) {
    print
   “How much unsolicited mail you like: <EM>”,
    $in{‘popupmenu’},“</EM>.”,“<p>”;
}

Working With Hidden Data Fields

You can store data in a hidden control using the <INPUT> tag by setting its TYPE attribute to hidden. Here’s how to create a hidden data field in lib1.cgi with the name hiddendata and the text Rosebud:

print
“<INPUT TYPE=\“hidden\” NAME=\“hiddendata\” VALUE=\“Rosebud\”>”;

Here’s how the code in lib2.cgi reads the text from the hidden control and displays it, as you saw earlier in Figure 19.4: 

if (&ReadParse(*in)) {
    print
   “The hidden data is <EM>”, $in{‘hiddendata’},“</EM>.”;
}

Creating Submit And Reset Buttons

To create a submit button, you can use the <INPUT> tag with the TYPE attribute set to submit; to create a reset button, you can use the <INPUT> tag with the TYPE attribute set to reset. Here’s how lib1.cgi creates both a submit and reset button:

print
“<CENTER>
<INPUT TYPE=\“submit\” NAME=\“submit\”>
<INPUT TYPE=\“reset\”>
</CENTER>”;

You can see the results in Figure 19.3, shown earlier in the chapter. When the user clicks the submit button, the data in the controls in the HTML form are sent to lib2.cgi; if he or she clicks the reset button, that data is reset.

Ending An HTML Form

To end an HTML form, you just print the </FORM> tag like this:

print
“</FORM>”;

When lib1.cgi prints </FORM>, that ends the form that groups all the HTML controls together. All that’s left is to end the HTML Web page itself—see the next topic.

Ending An HTML Document

To end an HTML document with cgi-lib.pl, you can use the HtmlBot subroutine. This subroutine just returns the string “</BODY>\n</HTML>\n”:

print &HtmlBot;

This is often the last line in a CGI script because it ends the Web page you’re creating. In our case, this is the last line in both lib1.cgi and lib2.cgi, and these scripts are ready to use.

Displaying All Variables

In this chapter, you’ve displayed the data values sent from HTML controls by explicitly printing these values out, but there’s an easier way to do it: You can use the PrintVariables subroutine if you don’t mind putting up with the formatting that subroutine creates. For example, if you replace lib2.cgi with this code, you’ll see how that formatting works when PrintVariables formats the data sent from the survey Web page:

#!/usr/local/bin/perl5

require ‘cgi-lib.pl’;

print &PrintHeader;
print &HtmlTop (“CGI Example Using cgi-lib.pl”);
if (&ReadParse(*in)) {
    print &PrintVariables;
}
print &HtmlBot;

You can see the result in Figure 19.5. All the data sent to lib2.cgi is, indeed, displayed, but without any explanatory text. In general, PrintVariables is good to help you debug your code, but not usually in the finished project.
