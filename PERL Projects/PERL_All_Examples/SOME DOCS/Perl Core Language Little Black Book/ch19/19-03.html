Centering HTML Elements

If you want to center HTML elements when creating them, just add a <CENTER> tag:

print
“<CENTER><H1>Here is the Survey!</H1></CENTER>
<H2>Please fill out our survey...</H2>”;

Creating A Bulleted List

To create the bulleted list that points out the advantages of filling out the survey to the user, just print the appropriate HTML directly: 

print
“Reasons for filling out our survey:<P>
<UL>
<LI>Fame</LI>
<LI>Fortune</LI>
<LI>Fun</LI>
</UL>”;

You can see the results in Figure 19.1, earlier in the chapter.

Creating A Hyperlink

To create a hyperlink, you just print the HTML for that hyperlink directly, as with other HTML elements. Here’s how to create the hyperlink to CPAN that appears in the survey page created by lib1.cgi: 

print
“If you would rather not fill out our survey, you might
be interested in
<A HREF=\“http://www.cpan.org/\”>CPAN</A>.”;

This hyperlink appears in Figure 19.1, shown earlier in the chapter.

Creating Horizontal Rules

As with other HTML elements, you create horizontal rules by printing out the required HTML directly. Here’s how to create a horizontal rule like the one in lib1.cgi: 

print
“<HR>”;

You can see this horizontal rule in Figure 19.1, shown earlier in the chapter.

Creating An HTML Form

The Perl CGI.pm package lets you use the startform method, but cgi-lib.pl doesn’t have a subroutine to match startform, which means you create an HTML form by yourself. The two attributes you should set in the <FORM> tag are METHOD (with the method of transferring data: POST or GET; you’ll use POST) and ACTION (which holds the URL of the CGI script you want to post data to). You can also indicate the encoding type for the form, although that’s not usually needed:

print
“<FORM METHOD=\“POST\”
ACTION=\“http://www.yourserver.com/user/cgi/lib2.cgi\”
ENCTYPE=\“application/x-www-form-urlencoded\”>”;

Now that we’ve placed a <FORM> tag in the Web page, all the following HTML controls will go into that form until we place a </FORM> tag in the page (see “Ending An HTML Form” later in the chapter).

Working With Text Fields

To create text fields, you use the <INPUT> tag, setting its TYPE attribute to text. Here’s how that works in lib1.cgi; you name the text field text and indicate that it should start empty by passing an empty string as its value:

print
“Please enter your name:
<INPUT TYPE=\“text\” NAME=\“text\” VALUE=\“\”><P>”;

This new text field appears in Figure 19.1, shown earlier in the chapter. Now that we have a text field, how do we read the data in that text field when it’s posted to lib2.cgi? See the next topic.

Reading Data From HTML Controls

To read the data from various HTML controls using cgi-lib.pl, you use the ReadParse subroutine, typically creating a hash named %in to hold the data from these controls. This hash is indexed by control name. For example, in the previous topic, we developed a text field named text; here’s how to create the %in hash and display the data from the text field:

require ‘cgi-lib.pl’;
if (&ReadParse(*in)) {
    print
   “Your name is: <EM>”, $in{‘text’},“</EM>.”,“<p>”;
}

The <EM> tags are just for emphasis, and display text in italics in most browsers. Note that we test ReadParse before using the %in hash—if ReadParse returns false, we don’t try to print out data from that hash.

That’s how you usually read data with cgi-lib.pl: call ReadParse to fill a hash and retrieve the data you want from that hash. You can see the previous code at work in Figure 19.4, shown earlier in the chapter.

Working With Text Areas

To create a text area, you use the <TEXTAREA> HTML tag. Here’s how lib1.cgi creates a text area to receive the user’s opinions in the Web survey; here, the text area is named textarea, has the default text “No opinion”, and has 10 rows and 60 columns:

print
“Please enter your opinion: <P><TEXTAREA NAME=\“textarea\”
ROWS=10 COLS=60>No opinion</TEXTAREA><P>”;

You can see the resulting text area in Figure 19.2, shown earlier in the chapter. When the user clicks the Submit Query button in the survey page, the data is posted to lib2.cgi; you display the contents of the text area this way, as shown earlier in the chapter in Figure 19.4: 

if (&ReadParse(*in)) {
    print
   “Your opinions are: <EM>”, $in{‘textarea’},“</EM>.”;
}

Working With Checkboxes

To create a checkbox group, you use <INPUT> tags, setting the TYPE attribute to checkbox and giving all the checkboxes the same name to group them. Here’s how lib1.cgi does it:

print
“Please indicate what products you use: <P>
<INPUT TYPE=\“checkbox\” NAME=\“checkboxes\”
VALUE=\“Shampoo\”>
Shampoo
<INPUT TYPE=\“checkbox\” NAME=\“checkboxes\”
VALUE=\“Toothpaste\”>
Toothpaste
<INPUT TYPE=\“checkbox\” NAME=\“checkboxes\” VALUE=\“Bread\”
CHECKED>Bread
<INPUT TYPE=\“checkbox\” NAME=\“checkboxes\” VALUE=
\“Cruise missiles\” CHECKED>Cruise missiles <P>”;

You can see the result in Figure 19.3, shown earlier in the chapter. Note that the user can select more than one checkbox in the checkbox group. When the data in this checkbox group is posted to lib2.cgi, $in{‘checkboxes’} will return a string with multiple values; to create a list from that string, you use cgi-lib.pl’s SplitParam this way:

if (&ReadParse(*in)) {
    print
   “You use these products: <EM>”,
    join(“,”, &SplitParam($in{‘checkboxes’})),“</EM>.”;
}

You can see the results in Figure 19.2, shown earlier in the chapter—note that all the user’s checkbox selections are displayed, separated by commas.

Working With Scrolling Lists

To create a scrolling list, you use the HTML <SELECT> tag. Here’s how I created a scrolling list in lib1.cgi to determine the user’s income level—note that you indicate what items should be in the list with the <OPTION> tag, and which one should be selected by default with the SELECTED attribute:

print
“Please indicate your income level: <P><SELECT
NAME=\“list\” SIZE=4>
<OPTION  VALUE=\“Highest\”>Highest
<OPTION SELECTED VALUE=\“High\”>High
<OPTION  VALUE=\“Medium\”>Medium
<OPTION  VALUE=\“Low\”>Low
</SELECT><P>”;

Note that you also specify the number of items to be displayed at once (that is, how high the control’s display should be, measured in lines of text) by setting the SIZE attribute. You can see the results in Figure 19.3, shown earlier in the chapter. Here’s how lib2.cgi reads the selection the user made and displays it, as shown earlier in Figure 19.4:

if (&ReadParse(*in)) {
    print
   “Your income level is: <EM>”,$in{‘list’},
   “</EM>.”,“<p>”;
}
